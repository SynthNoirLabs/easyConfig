# EasyConfig Project Context

## Project Overview
**EasyConfig** is a centralized Wails application designed to manage configuration files for various AI agents (Claude Code, Gemini CLI, Codex, Jules). It solves the problem of fragmented configuration ecosystems by providing a unified dashboard to find, edit, and extend agent capabilities, specifically focusing on Model Context Protocol (MCP) server injection.

## Technology Stack
- **Backend:** Go (v1.23+)
- **Application Framework:** Wails (v2.11.0)
- **Frontend:** React (v18), TypeScript, Vite
- **Configuration Formats:** JSON, TOML, YAML

## Directory Structure
- **`main.go`**: Application entry point.
- **`app.go`**: Main application logic and backend methods bound to the frontend.
- **`frontend/`**: Source code for the React application.
  - **`src/`**: UI components and logic.
  - **`wailsjs/`**: Auto-generated Go bindings for TypeScript.
- **`pkg/`**: Backend Go packages.
  - **`config/`**: Configuration handling types and logic (Providers, Services).
- **`build/`**: Application build artifacts and metadata.
- **`wails.json`**: Project configuration for Wails.

## Development & Build Commands

### Prerequisites
- Go 1.23+
- Node.js 18+
- Wails CLI installed (`go install github.com/wailsapp/wails/v2/cmd/wails@latest`)

### Running in Development Mode
To start the application with hot-reload:
```bash
wails dev
```

### Building for Production
To compile the application into a single binary:
```bash
wails build
```

### Frontend Development
The frontend can be managed independently for UI work:
```bash
cd frontend
npm install
npm run dev
```

## Architecture & Conventions

### Backend (Go)
- **Service-Oriented:** Logic is encapsulated in services (e.g., `pkg/config/service.go`) and exposed via `app.go`.
- **Configuration Management:** The app targets Global (user-home), Project (cwd), and Environment scopes.
- **Data Handling:** Robust marshaling/unmarshaling of multiple formats (JSON, TOML, YAML) is a core requirement.

### Frontend (React)
- **Component Library:** Uses standard React patterns.
- **Communication:** Interaction with Go backend is done via `window.go` bindings generated by Wails in `frontend/wailsjs/`.
